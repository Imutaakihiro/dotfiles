# Daily Report Formatter

文字起こしテキストを 5 セクションの日報フォーマットに整形し、Obsidian に保存する。

## ワークフロー

1. `date +%Y-%m-%d` で今日の日付を取得
2. 文字起こしテキストを解析して 5 セクションに分類
3. フォーマットに従って整形
4. `/Users/imutaakihiro/Projects/ObsidianImus/05_Journals/YYYY-MM-DD.md` に書き込み
5. 書き込んだファイルパスをユーザーに伝える
6. 保存完了後、`yoshida-shoin-fb` のワークフローをそのまま開始する（ユーザーへの確認不要）
   - 対象ファイルは直前に保存した日報ファイル
   - 「では、松陰先生に診ていただきます。」と一言添えて移行する

## セクション分類ルール

| セクション | 含める内容 |
|-----------|-----------|
| 今日の目標と達成状況 | 今日の目標・達成・未達・反省点 |
| 本日の業務内容 | 実際に行った業務・タスク・ミーティング（箇条書き） |
| 振り返り・自己分析 | 学び・気づき・考察・感想。テーマごとに `▼ テーマ名` で区切る |
| 先日のFBの返信 | 前回もらったフィードバックへの返答。なければ `（先日のフィードバックなし・割愛）` |
| 明日の目標 | 翌日の目標・ネクストアクション |

## 出力フォーマット

```
[挨拶文があれば先頭に1行。なければ省略]

【日報】YYYY-MM-DD
━━━━━━━━━━━━━━━━━━
:dart: 今日の目標と達成状況
━━━━━━━━━━━━━━━━━━
・[内容]

━━━━━━━━━━━━━━━━━━
:spiral_calendar_pad: 本日の業務内容
━━━━━━━━━━━━━━━━━━
・[内容]

━━━━━━━━━━━━━━━━━━
:mag: 振り返り・自己分析
━━━━━━━━━━━━━━━━━━
▼ [テーマ名]
・[内容]

━━━━━━━━━━━━━━━━━━
:speech_balloon: 先日のFBの返信
━━━━━━━━━━━━━━━━━━
[内容 or 「（先日のフィードバックなし・割愛）」]

━━━━━━━━━━━━━━━━━━
:pushpin: 明日の目標
━━━━━━━━━━━━━━━━━━
・[内容]
```

## ライティングルール

- 箇条書きは `-` ではなく `・` を使う
- 話し言葉を自然な文語体に変換する（「〜じゃないですか」→「〜である」など）
- 振り返りセクションは文字起こしに含まれているテーマのみ書く。捏造しない
- 明日の目標にサブ項目がある場合はインデント（全角スペース2個）で表現する
