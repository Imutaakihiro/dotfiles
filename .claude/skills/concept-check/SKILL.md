# コンセプトチェックスキル

## 概要

「SUPER CONCEPT」（著：パグ）のフレームワークに基づき、ユーザーのコンセプトを多角的に診断する。
フレームワークの詳細は `references/concept-framework.md` を参照すること。

**このスキルはループ型対話セッションである。** 1回のチェックで終わりではなく、改善→再チェックを繰り返す。

---

## ワークフロー

### フェーズ1：コンセプト受け取り

1. ユーザーからコンセプト文（テキスト直貼り or ファイルパス）を受け取る
2. ファイルパスが指定された場合は Read ツールで読み込む
3. `references/concept-framework.md` を読み込んでフレームワークを確認する
4. **コンセプトの種類を特定する**（事業 / 商品 / アカウントのどれか）

コンセプトの種類が不明な場合は、チェック開始前に1つだけ確認する。

---

### フェーズ2：診断レポートの出力

以下の4軸で診断し、**診断レポート**を出力する。

#### 軸1：基本4要素チェック

| 要素 | 含まれているか | 指摘 |
|------|--------------|------|
| 誰が（発信者の属性・実績） | ✅ or ❌ | |
| 誰に（ターゲット） | ✅ or ❌ | |
| どんな手段で | ✅ or ❌ | |
| どんな結果を（顧客が得るもの） | ✅ or ❌ | |

#### 軸2：良いコンセプトの公式チェック

**公式：目新しさ × 顧客の関心度 × 信頼**

各要素を 🔴弱い / 🟡普通 / 🟢強い の3段階で評価し、理由を添える。

#### 軸3：NGパターン診断

- [ ] ありきたりでないか（「月収100万円」「自動化で〜」など既視感がある表現）
- [ ] ユニークなだけでターゲットに刺さっていないか
- [ ] 世界観・実績の押し付けになっていないか
- [ ] 「この人って結局なんの人？」とならないか
- [ ] あれもこれも詰め込みすぎていないか

#### 軸4：コンセプトパーツ活用チェック

7つのパーツのうち、使えそうなものを提案する（`references/concept-framework.md` の「コンセプトパーツ7選」を参照）。

---

### 診断レポートのフォーマット

```
## コンセプト診断レポート

### 対象コンセプト
> [受け取ったコンセプト文をそのまま引用]

### 総合スコア
[S / A / B / C / D の5段階 + 一言評価]

### 軸1：基本4要素
[表形式で各要素の有無と指摘]

### 軸2：公式診断（目新しさ × 顧客の関心度 × 信頼）
[各3段階評価 + 理由]

### 軸3：NGパターン
[該当するNGパターンとその根拠]

### 軸4：強化できるパーツ提案
[使えそうなパーツと具体的な当てはめ案]

### 改善前後の比較
| | コンセプト |
|---|---|
| 改善前 | [元のコンセプト] |
| 改善後 | [改善案] |

### 次に取り組むべきこと
1. [最優先の改善点]
2. [次に着手すべきこと]
```

---

### フェーズ3：対話によるブラッシュアップ

診断レポート出力後、以下を問いかける：

```
---
【精度を上げるための確認事項】
1. 競合リサーチはできていますか？競合のコンセプト軸を教えてください。
2. ターゲットの生の声（悩み・言葉）は集めていますか？
3. あなた自身の差別化ポイント（実績・経験・特徴）は何ですか？
---
```

ユーザーの回答を受け取ったら、追加情報を踏まえてコンセプトをさらに磨き、改善案をアップデートする。
すべての改善ポイントが解消されるまで対話を繰り返す。

---

## 診断のポイント

- **顧客視点が最重要**：「自分が言いたいこと」ではなく「顧客がどこを重視するか」で判断する
- **絞りの確認**：ターゲット / 手段 / 結果のうち、どれか1つは必ず絞られているか確認する
- **改善案は具体的に**：「訴求を強くする」ではなく、実際に書き直した文章で示す
- **段階を意識する**：初期フェーズか成熟フェーズかでコンセプトの方向性が異なる
