# Obsidian Daily 日報整理スキル

今日の日記エントリー（05_Journals/YYYY-MM-DD.md）を読み込み、
**書き散らした内容を日報形式に整理して上書き保存する**スキル。

## 基本情報

- **Vault パス**: `/Users/imutaakihiro/Projects/ObsidianImus/`
- **日記の場所**: `05_Journals/YYYY-MM-DD.md`
- **01_Notes の3カテゴリ（これ以外のフォルダは作らない）**:
  - `01_Notes/Mind/` → マインド・マーケティング・ビジネス思考の学び（ファイル名: `Mind - タイトル.md`）
  - `01_Notes/Technical/` → 技術・プログラミング・AIの学び（ファイル名: `Technical - タイトル.md`）
  - `01_Notes/story/` → 体験談・具体的エピソード（ファイル名: `Story - タイトル.md`）

---

## 手順

### Step 1: 今日の日付と日記ファイルの読み込み

```bash
date '+%Y-%m-%d'
```

今日の日付（JST）を取得し、`05_Journals/YYYY-MM-DD.md` を読み込む。

- **ファイルが存在しない場合**: 「今日の日記がまだありません」とユーザーに伝えて終了する。
- **ファイルが存在する場合**: 内容を読み込み、Step 2 へ進む。

### Step 2: 内容を日報フォーマットに整理する

読み込んだ内容を以下の**日報フォーマット**に再構成する。

**ルール:**
- 内容を要約・改変・削除しない。元の言葉・表現をそのまま使う。
- 情報を適切なセクションに振り分けるだけ。
- 元にないセクションは省略する（空のセクションを作らない）。
- `- [ ] タスク` 形式のものは「アクションアイテム」セクションへ。

**日報フォーマット:**

~~~markdown
## 日報

### 本日の業務・活動
{今日やったこと・参加したこと・作業内容}

### 学び・気づき
{仕事・勉強・体験から得た学びや気づき}

### アクションアイテム
- [ ] {タスク}

### 振り返り・感想
{今日を振り返っての感想・反省・良かった点}
~~~

### Step 3: 整理した内容で上書き保存する

`05_Journals/YYYY-MM-DD.md` を、Step 2 で整理した日報フォーマットの内容で上書き保存する。

`Write` ツールを使う。

### Step 3.5: 関連 Note の自動紐づけ

日報の内容から**キーワードを抽出し、`01_Notes/` 全体をGrepして関連Noteを動的に発見**する。

#### 手順

1. **キーワード抽出**: 日報の学び・気づきセクションから主要な名詞・テーマを3〜5個抽出する。
   - 例: 「プレゼン」「リーダーシップ」「感情」「システム設計」「AI」など

2. **Grep検索**: 抽出したキーワードで `01_Notes/` 以下のファイル名を検索する。
   ```
   Glob pattern: 01_Notes/**/*.md
   キーワードがファイル名に含まれるものを優先してピックアップ
   ```

3. **関連度の判断**: 以下の基準でフィルタリングする。
   - ファイル名にキーワードが含まれる → 関連度：高
   - すでに日報内にwikiリンクがある → スキップ
   - README.md / template.md などのメタファイル → スキップ
   - 上限3〜5件に絞る

4. **日報末尾に追記する**: 関連Noteが1件以上見つかった場合のみ以下を追記する。

~~~markdown
---

## 関連 Note

- [[01_Notes/Mind/Mind - タイトル|タイトル]]
- [[01_Notes/Technical/Technical - タイトル|タイトル]]
~~~

**ルール:**
- 関連Noteが0件の場合はセクション自体を作らない。
- 確信度が低い（キーワードが弱い）場合はスキップする。無理に紐づけない。

### Step 4: 重要な学びを 01_Notes/ に展開するか提案する（任意）

整理した内容の中に「繰り返し参照したい学び・原則・エピソード」があれば、
以下のようにユーザーに提案する：

> 「『〇〇の学び』を 01_Notes/Mind/ にも残しておきますか？」

**カテゴリの判断基準:**
- マインド・マーケティング・ビジネス思考 → `01_Notes/Mind/Mind - タイトル.md`
- 技術・プログラミング・AI → `01_Notes/Technical/Technical - タイトル.md`
- 体験談・具体的エピソード → `01_Notes/story/Story - タイトル.md`
- **上記3つ以外のフォルダは絶対に作らない**

ユーザーが「はい」と言ったら、以下のフォーマットで保存する：

~~~markdown
# {タイトル}

{学びの内容を元の言葉のまま}

## 関連
- [[05_Journals/YYYY-MM-DD|YYYY-MM-DD の日報]]
~~~

そして、日報ファイルの該当箇所に `[[01_Notes/Mind/Mind - タイトル]]` の wiki リンクを追記する。

## 注意事項

- **内容は整理するだけ・要約・削除しない**。元の情報量を保つ。
- 空のセクションは作らない（内容があるセクションだけ出力する）。
- `01_Notes/` の展開先は Mind / Technical / Story の3つだけ。新フォルダを作ってはいけない。
- ファイルへの書き込みには `Write` ツールを使う。
